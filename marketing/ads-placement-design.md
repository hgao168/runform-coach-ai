## 一、广告位总览矩阵

```
┌──────────────────────────────────────────────────────────────────┐
│                   RunForm 全平台广告位布局                         │
│                                                                  │
│  平台         广告位              类型           优先填充          │
│──────────────────────────────────────────────────────────────────│
│  网站         首页 Hero 下方      原生卡片        自有产品 Pro      │
│  (movenova.ai) 产品页侧边栏      原生推广        自有产品 Pro      │
│               博客内嵌            AdSense        外部广告          │
│                                                                  │
│  iOS App      分析结果页底部       AdMob Banner   外部 + 自有       │
│               训练计划页          原生广告        自有产品 Pro      │
│                                                                  │
│  Android App  分析结果页底部       AdMob Banner   外部 + 自有       │
│               训练计划页          原生广告        自有产品 Pro      │
│                                                                  │
│  微信小程序   解锁高级分析        激励视频        外部广告          │
│               分析结果页底部       Banner 广告     外部广告          │
│               训练计划页          原生推广        自有 Pro          │
└──────────────────────────────────────────────────────────────────┘
```

### 核心设计原则

| 原则 | 说明 |
|------|------|
| 🛡️ **体验优先** | 广告不阻断核心流程，分析→报告→分享路径零干扰 |
| 🎯 **自有优先** | 有空位先填 Pro 推广，再填外部广告（填充率保障） |
| 🆓 **免费友好** | 首 3 天零广告体验期，Pro 用户永久无广告 |
| 📊 **数据驱动** | 每个广告位独立追踪展示/点击/收入，周级优化 |


| 规格 | 参数 |
|------|------|
| 位置 | 首页首屏 Hero CTA 正下方，距 Hero 底部 48px |
| 尺寸 | 响应式，max-width: 720px（桌面）/ 100vw-32px（移动） |
| 类型 | 原生推荐卡片（自有产品推广） |
| 频次控制 | 每次会话展示 1 次，关闭后本次会话不再出现 |
| 关闭按钮 | 右上角 ×，点击即永久隐藏（localStorage 记录） |
| 优先级 | **自有产品优先**，无 Pro 推广需求时替换为 AdSense 自适应广告 |
| 预估 CTR | 3-5%（自有产品强相关） |

### 2.2 产品页侧边栏

| 规格 | 参数 |
|------|------|
| 位置 | 功能描述页右侧（桌面端 sidebar），移动端置于内容底部 |
| 尺寸 | 300×250（桌面）/ 320×100（移动） |
| 类型 | 原生推广单元 —「RunForm Pro 功能清单 + CTA」 |
| Fallback | AdSense 300×250（自有推广优先级更高） |
| 展示条件 | 非登录用户或免费用户，Pro 用户不展示 |
| 预估值 | 展示量 ~1,000/月，CTR ~2% |

### 2.3 博客内嵌广告

| 规格 | 参数 |
|------|------|
| 位置 | 博客正文中段（文章滚动 50% 后插入），每篇文章 1 次 |
| 尺寸 | 自适应（推荐 728×90 桌面 / 320×100 移动） |
| 类型 | Google AdSense 自适应广告 |
| 频次控制 | 每文章 1 次（标准 AdSense 限制） |
| 广告内容审核 | 屏蔽竞品跑步 App（通过 AdSense 广告类别排除） |
| 预估收入 | 1,000 PV × $2 CPM = ~$2/月（起步阶段） |

## 四、Android App 广告位

> Android App 广告位布局与 iOS 一致，技术实现使用 Google AdMob Android SDK。
> 关键差异：Android 可增加 **插屏广告**（见 4.3），iOS 插屏因体验风险暂不启用。

### 4.1 分析结果页底部 Banner

与 iOS 完全一致：320×50 Banner，分析结果页底部固定，60s 自动刷新。

### 4.2 训练计划页原生广告

与 iOS 一致：第三位原生卡片，Pro 推广优先，AdMob Native 为 fallback。

### 4.3 Android 插屏广告（谨慎测试）

| 规格 | 参数 |
|------|------|
| 位置 | 查看历史记录列表 → 退回时弹出（返回路径上，非前进路径） |
| 类型 | AdMob Interstitial |
| 频次控制 | 每 3 次历史查看触发 1 次（3 次/日上限） |
| 展示条件 | 免费用户 + 超过 3 天保护期 |
| 关闭按钮 | 5 秒后可跳过（Google 规范） |
| 预估 eCPM | $2.00-5.00（插屏高于 Banner 3-5x） |
| 预估月收入 | DAU 200 × 0.3 触发率 × 30 天 × $3.00 eCPM ÷ 1000 = ~$5.40 |

### 6.1 广告位尺寸速查表

| 平台 | 广告位 | 尺寸 | 类型 |
|------|--------|------|------|

| 网站 | 博客内嵌 | 自适应 | AdSense |
| iOS | 分析页底部 | 320×50 / 智能横幅 | AdMob Banner |
| iOS | 训练计划 | Native 自适应 | 自有 / AdMob Native |
| Android | 分析页底部 | 320×50 / 智能横幅 | AdMob Banner |
| Android | 训练计划 | Native 自适应 | 自有 / AdMob Native |

### 6.2 频率控制总表

| 广告位 | 频次规则 | 技术实现 |
|--------|---------|---------|
| 网站 Hero 下方 | 每次会话 1 次 | localStorage `hero-ad-seen` flag |
| 网站侧边栏 | 每页 1 次 | 页面级渲染控制 |
| 博客内嵌 | 每文章 1 次 | AdSense 自动控制 |
| App 结果页 Banner | 每次分析 1 次 | 内存标记，分析完成重置 |
| App 训练计划 | 每会话 1 次 | `sessionStorage` flag |
| App 插屏 | 每 3 次历史查看 + 3 次/日上限 | App 内计数器 |
| 小程序激励视频 | 每次分析 1 次（用户主动） | 用户手动触发 |
| 小程序 Banner | 每页 1 次 | 微信自动刷新 30s |

## 九、风险与缓解

| # | 风险 | 概率 | 缓解 |
|---|------|------|------|
| R1 | 广告影响 App Store 评分（差评抱怨广告） | 中 | 3 天零广告保护期 + Pro 零广告 + 广告与内容区明确分隔 |
| R2 | AdMob 填充率低（跑步类 App 广告主少） | 中 | 自有推广优先填充空位；加入中介平台（AdMob Mediation） |
| R3 | 微信小程序广告审核不通过 | 低 | 提前提交审核，激励视频用官方组件无合规风险 |
| R4 | eCPM 远低于预期 | 中 | Week 2 上线后快速回收数据，A/B 测试广告位置 |
| R5 | 插屏广告导致 Android 卸载率上升 | 中 | 低频触发 + 仅放置在返回路径；Week 3 数据回收后决定是否保留 |
