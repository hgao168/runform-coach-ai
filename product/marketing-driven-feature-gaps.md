## 一、逐项审计

### 1. 骨架线叠加视频（核心视觉资产，P0）

**营销需求**：绿/红色骨架线随跑者摆动，错误时变红。8条短视频的主题涵盖跨步错误/驼背跑姿/着地方式/触地时间/步频对比等。

| 平台 | 当前状态 | 差距 | 调整建议 |
|------|---------|------|---------|
| **iOS** | ✅ `LiveGuidanceRecorderView` 录制时叠加 Vision 骨骼线。但骨骼线是白色/绿色单一颜色，不会因错误变红。**用户无法导出带骨架线的视频** — 只有实时预览，录制结果是原始视频。 | 骨架线无错误着色；无导出功能 | ① 在 PoseExtractor 分析结果中标记异常关节角度（膝超伸/髋下沉/躯干前倾>20°），给骨架线着色（绿正常/红异常）；② 新增 `exportOverlayVideo` 功能：将分析结果中的骨架线渲染叠加到原始视频帧上导出 |
| **Android** | ⚠️ `LiveGuidanceRecorderScreen` 有 CameraX + ML Kit 实时骨骼线。同样无错误着色、无导出。 | 同 iOS | 同 iOS |
| **WeChat** | ❌ 无实时骨架线（平台限制）。视频上传后后端返回分析结果，但无视觉叠加。 | 无法实现实时，但可做静态对比图 | Canvas 渲染「分析前 vs 分析后」骨架线静态对比图（已有 share-card.js 基础） |
| **Web** | ❌ 无骨架线功能 | 不适用 | 放 iOS/Android 的骨架线视频截图作为营销素材即可 |

**产品调整**：
- **P0**：iOS + Android 分析结果页增加「骨架线叠加回放」— 用户查看分析结果时，播放原始视频并在上方叠加骨架线（绿色正常/红色异常关节），支持截屏分享
- **P1**：iOS + Android 增加「导出骨架线视频」— 将叠加后的视频保存到相册
- **优先级判断**：如果截屏分享足够用于前8条短视频素材，导出功能可降 P1

### 2. 「旧片考古」社群裂变活动

**营销需求**：用户上传旧比赛/练跑影片 → AI分析 → 分享分析结果到社媒（带「旧片考古」标签）→ 免费解锁下月课表。

| 平台 | 当前状态 | 差距 | 调整建议 |
|------|---------|------|---------|
| **视频上传分析** | ✅ 已支持相册选择 + 录制上传 | — | 无 |
| **分析结果分享** | ✅ iOS ShareSheet / Android ShareSheet / WeChat 分享卡片 | 分享文案未匹配「旧片考古」叙事 | 在分享卡片增加「旧片考古」主题模板：标题「我在XX比赛的跑姿被AI考古了」，副标题「第30公里跑姿崩坏——Runform发现了我的致命问题」 |
| **解锁课表奖励** | ❌ 不存在 | 无「分享→解锁」联动机制 | 最简单实现：分享后返回 App，本地标记 `hasSharedOldVideo=true`，解锁训练计划中下一周的课表（前端状态切换即可，不需后端） |
| **活动专属入口** | ❌ 无 | 无活动页 | WeChat/Web 新增「旧片考古」活动页：上传入口 + 已参与用户的分享墙 |

**产品调整**：
- **P1**：分析结果分享卡片增加「旧片考古」模板（仅前端文案调整，1-2 SP）
- **P2**：分享后解锁课表（前端状态标记，1 SP）
- **P2**：活动专属页面（Web 或 WeChat，3 SP）

### 3. 「无痛5K挑战赛」种子用户

**营销需求**：招募20位新手，4周养成班。每周一上传跑姿影片，App根据跑姿调整下周训练量（如跑姿走样则降低跑量+加入防伤训练）。

| 功能 | 当前状态 | 差距 | 调整建议 |
|------|---------|------|---------|
| **5K训练计划** | ✅ 已存在（iOS/Android/WeChat/Web） | 计划是静态课表，非动态调整 | 在训练计划周视图增加「本周跑姿复检提醒」入口；复检后如跑姿评分下降，在计划中插入「防伤训练日」（臀肌激活/踝关节稳定） |
| **周跑姿复检** | ❌ 无 | 无周期性复检机制 | 训练计划中每周一推送「本周跑姿复检」；用户上传影片后对比上周跑姿评分 |
| **教练面板** | ✅ RF-602 已完成 | 教练可查看学员跑姿概览 | 用于管理20位种子用户 |
| **跑姿驱动的训练调整** | ❌ 无 | 分析和训练计划独立 | ① 分析结果页增加「基于你的跑姿，建议在训练计划中加入：XXX训练」提示；② 训练计划生成时读取最近跑姿数据，自动插入针对性训练 |

**产品调整**：
- **P1**：训练计划周视图增加「跑姿复检」入口 + 复检后跑姿评分对比（iOS/Android，5 SP）
- **P2**：跑姿驱动的训练计划动态调整（后端 + 各端，8 SP，可延至 Sprint 6）

### 4. 「检测→训练→修正」闭环

**营销需求**：分析发现核心无力导致骨盆下沉 → 训练计划自动加入10分钟臀肌激活训练 → 「这是根据你身体密码量身打造的防伤完赛计划」。

| 平台 | 当前状态 | 差距 | 调整建议 |
|------|---------|------|---------|
| **分析结果 → 训练建议** | ⚠️ 分析结果有「训练建议」文字，但未联动训练计划 | 建议是文字提示，未实际修改课表 | 在分析结果页「训练建议」区增加「应用到此周训练计划」按钮，点击后在训练计划中插入建议的训练日 |
| **营销文案** | ⚠️ 当前文案偏技术向（「步频168 SPM，偏低」） | 缺少「防伤」「量身打造」「身体密码」等叙事 | 调整分析结果页的 summary 文案模板（仅文本，前端改）：从「步频偏低，建议加快到170」→「我们发现你的步频偏低，这可能增加膝盖冲击。你的专属5K计划已自动加入步频训练日」 |

**产品调整**：
- **P1**：分析结果页训练建议文案调整（前端文案改动，0.5 SP/端）
- **P2**：分析结果→训练计划联动（3 SP）

### 5. 伤病 SEO 截流着陆页

**营销需求**：Google/百度搜索「跑步膝蓋痛怎麼辦」→ 着陆页 → App 下载 CTA。

| 平台 | 当前状态 | 差距 | 调整建议 |
|------|---------|------|---------|
| **Web 着陆页** | ⚠️ movenova.ai 有 features 页、首页，无伤病专题页 | 无 SEO 优化的伤病科普页 | 新建 Web 着陆页 `/injury-prevention`（或类似路由），覆盖 5 个高频伤病关键词：膝盖痛/脚底痛/小腿痛/髋部痛/腰酸。每页含：原因科普 + Runform 如何检测 + CTA（下载/小程序码）。SEO meta tags 完善 |
| **结构化数据** | ❌ 无 | 无 FAQ/HowTo schema | 着陆页增加 FAQ 结构化数据 |

**产品调整**：
- **P1**：Web 伤病着陆页 x5（5 SP）
- **P2**：SEO 结构化数据（1 SP）

## 二、优先级排序

| 优先级 | 调整项 | 平台 | SP | 阻塞什么 |
|:------:|--------|------|:--:|---------|
| **P0** | 骨架线错误着色 + 截屏分享 | iOS + Android | 5 | 阻塞 8 条骨架线视频素材制作 |
| **P1** | 分析结果分享卡片「旧片考古」模板 | iOS + Android + WeChat | 2 | 阻塞「旧片考古」活动社媒传播 |
| **P1** | 伤病 SEO 着陆页 x5 | Web | 5 | 阻塞伤病搜索截流 |
| **P1** | 各端 onboarding/首页防伤文案 | 全平台 | 2 | 阻塞品牌定位落地 |
| **P1** | 分析结果文案调整（防伤叙事） | iOS + Android | 1 | 阻塞「检测→训练」叙事 |
| **P1** | 训练计划增加「跑姿复检」入口 | iOS + Android | 5 | 阻塞「无痛5K」周复检流程 |
| **P2** | 骨架线视频导出 | iOS + Android | 3 | 非阻塞（截屏可替代） |
| **P2** | 分享后解锁课表 | WeChat | 1 | 非阻塞（手动发码也可） |
| **P2** | 分析→训练计划联动 | 后端 + 各端 | 3 | 延至 Sprint 6 |
| **P2** | 「旧片考古」活动专属页 | Web 或 WeChat | 3 | 现阶段社媒帖子可替代 |
| **P2** | SEO 结构化数据 | Web | 1 | 非阻塞 |

## 三、总结

**必须做（P0+P1）**：7 项，20 SP
- 骨架线着色 + 截屏（iOS/Android）
- 分享卡片「旧片考古」模板
- Web 伤病着陆页
- 防伤文案调整（全平台）
- 训练计划跑姿复检入口

**可延后（P2）**：5 项，11 SP

**不需要改的**：邀请码系统、挑战赛、教练面板、UGC入口、里程碑庆祝图、推送、分享图服务 — 这些 Sprint 5 已完成的功能可以直接服务于修订后的营销方案。
