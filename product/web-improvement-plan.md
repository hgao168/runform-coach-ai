## Day 1：导航重构 + 上传流程优化

**当前问题：** Features, Analysis, History, Challenge, Training Plan, Compare 作为顶级菜单项，用户无法理解它们是 RunForm 的子功能。

**目标结构：**

```
Products ▼
  ├── RunForm      (跑步姿态分析)
  │     ├── Features        (功能总览)
  │     ├── Analysis        (分析报告)
  │     ├── History         (历史记录)
  │     ├── Challenge       (挑战)
  │     ├── Training Plan   (训练计划)

  ├── TennisForm   (网球姿态分析)
  ├── GolfForm     (高尔夫姿态分析)

```

### 1.2 上传 / 分析流程清晰化

**当前问题：** 用户不确定该上传什么视频、什么角度、需要多久、能获得什么结果 —— 不确定性导致放弃。

**新增内容：**

1. **视觉化 3 步引导模块**（上传按钮上方）
   - 第 1 步：拍摄 8-10 秒跑步视频（配图：手机拍摄示意）
   - 第 2 步：上传侧面/正面/背面视角（配图：三个角度示意）
   - 第 3 步：30 秒内获取 AI 生物力学报告（配图：报告预览）

2. **示例演示视频**
   - 嵌入一段"好的上传 vs 差的上传"对比视频
   - 标注：正确角度、光线条件、距离要求

3. **进度条 + 等待时间提示**
   - 上传进度条（百分比）
   - 分析阶段：骨架动画 + "预计还需 30 秒"
   - 完成后：渐变过渡到报告页面

4. **格式与要求说明**
   - 支持格式：MP4, MOV
   - 建议分辨率：1080p+
   - 文件大小：< 200MB

## Day 2：AI 透明度 + 信任建设

### 2.1 "AI 如何分析你的跑步姿态"页面

**目标：** 消除用户对 AI 分析准确性和隐私的疑虑，尤其是欧洲市场（GDPR）。

**页面内容结构：**

```
/ai-transparency
├── 1. 姿态检测原理
│     - 计算机视觉 + BlazePose/MediaPipe 关键点检测
│     - 逐帧分析：髋、膝、踝、肩关节角度
│     - 运动轨迹追踪：步幅、触地时间、垂直振幅
│
├── 2. 置信度分数解读
│     - 0-100 分含义：角度清晰度、光线质量、遮挡情况
│     - 低分（<60）：建议重新拍摄
│     - 中分（60-85）：结果可靠，个别指标需注意
│     - 高分（85+）：分析精度高
│
├── 3. 分析局限性（诚实透明）
│     - 非医疗诊断工具
│     - 无法替代专业教练/物理治疗师
│     - 相机角度偏差可能影响结果
│     - 始终建议结合专业意见
│
├── 4. 方法论背书
│     - 运动科学参考文献
│     - 教练审核机制
│     - 持续迭代更新
│
├── 5. 隐私与安全保障
│     - 视频分析后自动删除（如果是真的）
│     - GDPR 合规声明
│     - 数据加密传输
│     - 永远不会分享或出售视频数据
│
└── 6. 常见问题 FAQ
      - 需要什么设备？
      - 室内还是室外拍摄？
      - 可以分析哪些运动？
```

| 上午 09:00-10:30 | 撰写 AI 透明度页面完整文案 | 文案定稿 |
| 上午 10:30-12:00 | 页面设计 + 前端开发 | 页面组件 |
| 下午 13:00-14:00 | 隐私声明页面（GDPR 合规） | /privacy |

## Day 5：移动端高级感 UI 升级

### 5.1 视觉语言升级

**目标：** 让 UI 从"早期创业 MVP"升级到"精品运动科技平台"级别，对标 Strava / Whoop / Nike Run Club。

**改进清单：**

| 维度 | 现状 | 目标 |
|------|------|------|
| 配色 | — | 深色渐变背景 + 高对比度运动色点缀 |
| 字体 | — | 更大字号层级、运动感标题字体 |
| 动效 | 少/无 | 滚动视差、卡片渐入、数字递增动画 |
| 卡片 | 扁平 | 毛玻璃效果、微阴影、圆角卡片 |
| 按钮 | — | 渐变 CTA 按钮、触控反馈 |
| 间距 | 密集文字 | 充分留白、大字引领 |

**具体实施方案：**

1. **全局配色系统**
   - 主背景：深炭灰到黑色渐变（`#1a1a2e → #0f0f1a`）
   - 品牌色：活力橙/电光蓝作为 CTA 和强调色
   - 卡片：半透明毛玻璃效果（`backdrop-filter: blur`）

2. **排版系统**
   - H1: 48px bold（移动端 36px）
   - H2: 32px semibold（移动端 28px）
   - 正文: 18px（移动端 16px）
   - 数字指标：等宽大字（如 Strava 的统计数据展示）

3. **动效系统**
   - 页面滚动：视差效果（hero 区域）
   - 卡片：`fadeInUp` 渐入动画（Intersection Observer）
   - 数字：计数递增动画（步频、速度、分数）
   - 上传进度：脉冲光环动画
   - 导航切换：平滑过渡

4. **移动端专项优化**
   - 底部导航栏（拇指热区友好）
   - 卡片滑动手势
   - 更大的触摸目标（44pt 最小点击区域）
   - Apple Health 风格圆环图（步频、对称性指标）

| 时间 | 任务 | 产出 |
|------|------|------|
| 上午 09:00-10:30 | 建立全局设计 Token（颜色、间距、字体、阴影） | Design tokens |
| 上午 10:30-12:00 | 全局排版 + 配色系统应用到所有页面 | 视觉一致性 |
| 下午 13:00-15:00 | 动效系统开发（滚动视差、卡片渐入、数字动画） | 动效就绪 |
| 下午 15:00-16:30 | 移动端专项优化（底部导航、手势、触摸目标） | 移动端体验 |
| 下午 16:30-17:30 | Apple Health 风格圆环图组件 + 数据可视化升级 | 图表组件 |
| 晚间 17:30-19:00 | 全站响应式回归测试 + 性能审计（Lighthouse） | 质量保证 |

**Day 5 交付物：** 全面视觉升级、动效系统、移动端体验优化、性能审计报告。

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 导航重构导致 SEO 损失 | 流量下滑 | 保持 URL 不变，301 重定向，提交新 sitemap |
| 案例素材不足 | Day 3 无法交付 | 先上 2 个案例，标注"更多案例即将上线" |
| 动效影响移动端性能 | 卡顿跳出 | 低端设备降级方案，`prefers-reduced-motion` 支持 |
| 新建页面收录慢 | SEO 效果延迟 | 提交 sitemap，主动 Google Search Console 索引请求 |
